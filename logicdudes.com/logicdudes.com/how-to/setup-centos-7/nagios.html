Setup Nagios

Objective

Install and configure Nagios on CentOS to monitor basic server status and report errors via email and text message

1. Install Nagios

yum install nagios

Nagios is available via the EPEL repository and can be installed with yum. You will notice that httpd and php are included in the dependencies and will also be installed.


2. Install Nagios Plugins

yum install nagios-plugins-http \
nagios-plugins-load \
nagios-plugins-users \
nagios-plugins-ping \
nagios-plugins-disk \
nagios-plugins-ssh \
nagios-plugins-swap \
nagios-plugins-procs

These are the plugins necessary to use the default server checks. Run yum search nagios-plugins to see a list of more. Nagios plugins are stored in /usr/lib64/nagios/plugins/

3. Add Contacts

vim /etc/nagios/objects/contacts.cfg

Replace the email address in the define contact block with your own. If you would like, add create another contact block to include another email address (like one that would text your phone ;)

3. Create Web Password

htpasswd -c /etc/nagios/passwd nagiosadmin

This will create a password for the username nagiosadmin that will be used to access the web interface. Do not change the username because it is linked to other aspects of Nagios.

5. Secure Virtual Host

We’re going to restrict the Nagios website, making it only accessible to your IP and over the port 9000.

vim /etc/httpd/conf.d/nagios.conf

Change the Apache Virtual Host configuration file so only your IP address is allowed to access the interface. Change the the Order and Allow lines to Order deny,allow and Allow from <your IP> in both parts of the file.

 

 

vim /etc/httpd/conf/httpd.conf

Chang the port that Apache will listen on by changing the Listen directive to Listen 9000

 

6. Add Firewall Policy

vim /etc/sysconfig/iptables

-A INPUT -p tcp -m tcp –dport 80 -j ACCEPT

Edit the iptables configuration file and allow tcp traffic on port 80 (you can just uncomment the line we created in the previous tutorial). Once you have added the rule, restart iptables to put the new configuration into effect

systemctl restart iptables


6. Start the Services

systemctl start httpd && systemctl enable httpd

systemctl start nagios && systemctl enable nagios

7. Login to Nagios

http://<your ip>:9000/nagios

Go to the address and enter the username: nagiosadmin and the password that you setup to access the web interface.

8. Check Services

Click on the Services link in the Current Status menu to see your server’s current status. These checks are performed every five minutes by default. You can schedule a check by clicking Re-schedule the next check of this service


 

Don’t worry about the HTTP warning; it’s there because Nagios is looking on port 80 for a webserver but we configured Apache (httpd) to listen on port 9000. The error will go away later once we have installed NginX.

That’s It!

Now you can monitor your server and receive emails alerts about major errors
